# Quickstart

Get CHIP8-rs running in just a few steps. This guide will walk you through creating a basic CHIP-8 emulator instance, loading a ROM, and executing it.

## Prerequisites

Make sure you have Rust installed. If not, get it from [rustup.rs](https://rustup.rs/).

<Steps>
  <Step title="Create a new Rust project">
    ```bash
    cargo new my-chip8-emulator
    cd my-chip8-emulator
    ```
  </Step>

  <Step title="Add CHIP8-rs as a dependency">
    Add this to your `Cargo.toml`:
    
    ```toml
    [dependencies]
    chip8-rs = { git = "https://github.com/Ferryistaken/CHIP8-rs" }
    ```
  </Step>

  <Step title="Create and run your first emulator">
    Replace the contents of `src/main.rs` with this basic example:

    ```rust
    use chip8_rs::chip8::Chip8;
    use std::path::PathBuf;
    use std::{thread, time};

    fn main() {
        // Create a new CHIP-8 instance
        let mut chip8 = Chip8::new();
        
        // Load a ROM file
        let rom_path = PathBuf::from("path/to/your/rom.ch8");
        chip8.load_rom(rom_path).expect("Failed to load ROM");
        
        // Run the emulator with a 10ms cycle delay (~100Hz)
        let cycle_wait_time = time::Duration::from_millis(10);
        
        loop {
            chip8.Cycle();
            chip8.pretty_print_video();
            thread::sleep(cycle_wait_time);
        }
    }
    ```

    This creates an emulator instance, loads a ROM, and runs the main execution loop with built-in terminal rendering.
  </Step>

  <Step title="Get a test ROM">
    Download a test ROM to try out your emulator. The repository includes several test ROMs:
    
    - `test_opcode.ch8` - Opcode test suite
    - `IBM-Logo.ch8` - Displays the IBM logo
    - `BC_test.ch8` - Comprehensive test

    Or find public domain ROMs online to test with.
  </Step>

  <Step title="Run it">
    ```bash
    cargo run
    ```
    
    You should see the emulator running in your terminal!
  </Step>
</Steps>

## Using the Built-in CLI

CHIP8-rs comes with a fully-featured CLI emulator using the `cli` feature:

```toml
[dependencies]
chip8-rs = { git = "https://github.com/Ferryistaken/CHIP8-rs", features = ["cli"] }
```

Then you can use the `cli` binary:

```rust
use chip8_rs::chip8::Chip8;
use std::path::PathBuf;

fn main() {
    let mut chip8 = Chip8::new();
    chip8.load_rom(PathBuf::from("roms/test.ch8")).unwrap();
    chip8.play(10); // 10ms per cycle
}
```

Or build and run the CLI directly:

```bash
git clone https://github.com/Ferryistaken/CHIP8-rs
cd CHIP8-rs
cargo build --release --features cli
./target/release/cli --rom roms/test_opcode.ch8 --clock 10
```

## Common Patterns

### Custom Rendering

If you want custom rendering instead of the built-in terminal output:

```rust
use chip8_rs::chip8::Chip8;

fn main() {
    let mut chip8 = Chip8::new();
    chip8.load_rom("rom.ch8".into()).unwrap();
    
    loop {
        chip8.Cycle();
        
        // Access the raw video buffer
        let video = chip8.export_video();
        
        // video is a &[u32; 2048] (64×32 pixels)
        // 0xFFFFFFFF = white pixel, 0x00000000 = black pixel
        render_custom(video);
        
        std::thread::sleep(std::time::Duration::from_millis(10));
    }
}

fn render_custom(video: &[u32; 2048]) {
    // Your custom rendering logic here
    // Could be SDL2, minifb, pixels, or any graphics library
}
```

### Handling Keyboard Input

```rust
use chip8_rs::chip8::Chip8;

fn main() {
    let mut chip8 = Chip8::new();
    chip8.load_rom("rom.ch8".into()).unwrap();
    
    loop {
        chip8.Cycle();
        
        // Handle input from your input system
        // CHIP-8 uses a 16-key hexadecimal keypad (0-F)
        if key_pressed(Key::Num1) {
            chip8.key_down(0x1);
        }
        if key_released(Key::Num1) {
            chip8.key_up(0x1);
        }
        
        // Render and delay...
    }
}
```

<Note>
The keyboard layout typically maps to:
```
1 2 3 C    →    1 2 3 4
4 5 6 D    →    Q W E R  
7 8 9 E    →    A S D F
A 0 B F    →    Z X C V
```
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="ROM doesn't load">
    Make sure:
    - The ROM file path is correct
    - The ROM is a valid CHIP-8 binary (.ch8 file)
    - The ROM size is within memory limits (typically under 3.5KB)
  </Accordion>

  <Accordion title="Display issues">
    The built-in `pretty_print_video()` clears the terminal on each frame. If you see flickering:
    - Try increasing the cycle delay
    - Use a custom rendering solution
    - Implement double-buffering in your terminal
  </Accordion>

  <Accordion title="No feature 'cli' found">
    The CLI feature requires additional dependencies. Make sure you're using:
    ```toml
    chip8-rs = { git = "...", features = ["cli"] }
    ```
    Or clone and build from source with `cargo build --features cli`
  </Accordion>
</AccordionGroup>
