## Overview

The `Chip8` struct is the core data structure that represents the entire CHIP-8 virtual machine state. It contains all registers, memory, timers, and display state needed to emulate the CHIP-8 system.

## Definition

Defined in `src/chip8.rs:18-39`

```rust
pub struct Chip8 {
    registers: [u8; 16],
    memory: [u8; 4096],
    index_register: u16,
    program_counter: u16,
    stack: [u16; 16],
    stack_pointer: u8,
    delay_timer: u8,
    sound_timer: u8,
    pub keypad: [u8; 16],
    recent_presses: VecDeque<u8>,
    pub video: [u32; 64 * 32],
    op_code: u16,
    table: [fn(&mut Chip8); 0xF+1],
    table0: [fn(&mut Chip8); 0xE+1],
    table8: [fn(&mut Chip8); 0xE+1],
    tableE: [fn(&mut Chip8); 0xE+1],
    tableF: [fn(&mut Chip8); 0x65+1],
    debug_mode: bool,
    last_opcode: u16,
    rng_state: u32,
}
```

## Fields

### Private Fields

<ParamField path="registers" type="[u8; 16]">
  16 general-purpose 8-bit registers (V0-VF). Register VF is used as a flag register for some operations.
</ParamField>

<ParamField path="memory" type="[u8; 4096]">
  4KB of RAM. Addresses 0x000-0x1FF are reserved for the interpreter (including font data at 0x050-0x0A0), and program ROM starts at 0x200.
</ParamField>

<ParamField path="index_register" type="u16">
  The index register (I), used to store memory addresses for operations.
</ParamField>

<ParamField path="program_counter" type="u16">
  The program counter (PC), points to the current instruction in memory. Initialized to 0x200.
</ParamField>

<ParamField path="stack" type="[u16; 16]">
  The call stack, holds up to 16 return addresses for subroutine calls.
</ParamField>

<ParamField path="stack_pointer" type="u8">
  Points to the top of the stack.
</ParamField>

<ParamField path="delay_timer" type="u8">
  8-bit delay timer that decrements at 60Hz when non-zero.
</ParamField>

<ParamField path="sound_timer" type="u8">
  8-bit sound timer that decrements at 60Hz. A beep sound should play when non-zero.
</ParamField>

<ParamField path="op_code" type="u16">
  The currently executing opcode.
</ParamField>

<ParamField path="table" type="[fn(&mut Chip8); 0xF+1]">
  Main opcode dispatch table mapping the first nibble to handler functions.
</ParamField>

<ParamField path="table0" type="[fn(&mut Chip8); 0xE+1]">
  Dispatch table for opcodes starting with 0x0.
</ParamField>

<ParamField path="table8" type="[fn(&mut Chip8); 0xE+1]">
  Dispatch table for opcodes starting with 0x8 (arithmetic/logic operations).
</ParamField>

<ParamField path="tableE" type="[fn(&mut Chip8); 0xE+1]">
  Dispatch table for opcodes starting with 0xE (keyboard operations).
</ParamField>

<ParamField path="tableF" type="[fn(&mut Chip8); 0x65+1]">
  Dispatch table for opcodes starting with 0xF (misc operations).
</ParamField>

<ParamField path="debug_mode" type="bool">
  When true, enables debug output to stderr for opcode execution.
</ParamField>

<ParamField path="last_opcode" type="u16">
  Stores the most recently executed opcode, accessible via `last_opcode()` method.
</ParamField>

<ParamField path="rng_state" type="u32">
  Internal state for the pseudo-random number generator used by the CXKK opcode.
</ParamField>

### Public Fields

<ParamField path="keypad" type="[u8; 16]" public>
  16-key hexadecimal keypad state (0x0-0xF). Each element is 0 when key is released, 1 when pressed.
</ParamField>

<ParamField path="recent_presses" type="VecDeque<u8>">
  Queue of recent key press events, used for the FX0A "wait for key" opcode.
</ParamField>

<ParamField path="video" type="[u32; 64 * 32]" public>
  64x32 monochrome display buffer. Each u32 is either 0x00000000 (off) or 0xFFFFFFFF (on).
</ParamField>
